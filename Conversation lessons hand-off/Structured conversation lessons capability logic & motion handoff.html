Animation Library — Motion (motion.dev)
All animations in this handoff use Motion's animate() API — a lightweight, production-grade JavaScript animation library. Here's what you need to know to get started:
  • Install: npm install motion — then import { animate } from "motion"
  • Core pattern: animate(from, to, { type: "spring", stiffness, damping, onUpdate }) — drives a value between two numbers using spring physics, calling onUpdate every frame
  • Completion: animate(...).then(() => { }) — returns a promise that resolves when the animation finishes
  • Keyframes: animate(0, [4, -4, 3, -3, 0], { duration: 0.5 }) — pass an array as the to value to animate through multiple waypoints
  • Duration: Always in seconds (not milliseconds) — e.g. 0.5 = 500ms
  • Docs: animate() · React integration · Quick start

Chip Entrance Animation

When a chip moves into the sentence box, it plays a combined scale + slide-up spring animation for a tactile "landing" feel.

// Chip starts invisible, offset 8px below, at 92% scale chip.style.opacity = "0"; chip.style.transform = "translateY(8px) scale(0.92)"; animate(0, 1, { type: "spring", stiffness: 600, damping: 24, onUpdate(v) { chip.style.transform = `translateY(${(1 - v) * 8}px) scale(${0.92 + v * 0.08})`; chip.style.opacity = `${v}`; } }).then(() => { chip.style.opacity = "1"; chip.style.transform = ""; });

Interaction Logic

Tap a word in the bank → it moves into the sentence box (ghost left behind, chip entrance animation). Tap a chip in the sentence box → it returns to the word bank. Submit is shown when the sentence box has at least one chip; hidden when empty. Submit triggers correct or incorrect answer animation. After incorrect, chips reset back to the word bank and placeholder shows "Try again..." Skip button is hidden for the first two attempts. On the 3rd incorrect attempt, Skip appears below the word bank. Skip hides again whenever Submit is visible (sentence box has chips). Tapping Skip resets everything and advances to the next question.

Input Behavior — Critical Rules

The matching exercise must feel fast and responsive. There is no global input lock at any point. The user should be able to tap as fast as they want. Here are the specific rules:

ScenarioBehavior
Correct match Animation plays immediately, no delay. Both buttons become non-interactive (pointerEvents: none) and stay in the correct (green) state. Input is never blocked — user can immediately tap the next pair while the correct animation is still playing.
Incorrect match Both buttons jiggle (orange, 500ms rotation). Input is never blocked — user can tap other buttons immediately. The two jiggling buttons themselves are untappable while jiggling (they ignore clicks while in state-incorrect).
Tap a jiggling button Ignored. Buttons with state-incorrect class reject clicks. Once the jiggle ends and the button resets to default, it becomes tappable again.
Tap a correct (green) button Ignored. Already-matched buttons reject clicks permanently.
Tap audio button (left) Plays the audio clip for that word immediately. Audio cannot be cancelled by the user — it plays to completion. However, tapping a different audio button instantly stops the current audio and plays the new one instead. This is the only way to interrupt playback.
Switch left selection User can freely re-tap a different left button at any time — the previous left selection deselects. If the new button is an audio button, its audio plays immediately (interrupting any currently playing audio).
Tap right before selecting left Ignored — must pick left first.
Left button started jiggling after selection If the previously selected left button enters state-incorrect (from a prior incorrect pair), the selection is cleared. User must pick a new left button.

Key differences from multi-choice: (1) after incorrect jiggle, matching pair buttons reset back to default (white) — they do not go to disabled (gray). (2) No input locking at any point — all animations are non-blocking.

Audio Playback Rules

Each left-column button plays a TTS audio clip for its word. The implementation should keep a single shared Audio instance (or equivalent) so that only one clip plays at a time.

ActionBehavior
Tap audio buttonPlay the audio clip immediately. Select the button.
Tap same audio button againReplay from the beginning. Selection stays.
Tap different audio buttonStop current audio instantly, play the new clip. Selection moves to the new button.
Audio finishes naturallyNothing happens — button stays selected, waiting for a right-column tap.
User taps right column while audio is playingAudio continues playing (do not stop it). Pair evaluation proceeds normally.

9. React Component

Ready-to-use React component implementing all states and both animations.

AnimatedTapButton.tsx
import React, { useRef, useCallback } from "react"; import { animate } from "motion"; type ButtonState = "default" | "pressed" | "selected" | "correct" | "incorrect" | "disabled"; interface Props { label: string; state: ButtonState; onTap?: () => void; } export function AnimatedTapButton({ label, state, onTap }: Props) { const ref = useRef<HTMLButtonElement>(null); const animateCorrect = useCallback(() => { const el = ref.current; if (!el) return; animate(1.0, 1.08, { type: "spring", stiffness: 600, damping: 18, onUpdate(v) { el.style.transform = `scale(${v})`; } }).then(() => { animate(1.08, 1.0, { type: "spring", stiffness: 500, damping: 20, onUpdate(v) { el.style.transform = `scale(${v})`; } }); }); }, []); const animateIncorrect = useCallback(() => { const el = ref.current; if (!el) return; animate(0, [4, -4, 3, -3, 2, -2, 0], { duration: 0.5, onUpdate(v) { el.style.transform = `rotate(${v}deg)`; } }).then(() => { el.style.transform = "rotate(0deg)"; }); }, []); React.useEffect(() => { if (state === "correct") animateCorrect(); if (state === "incorrect") animateIncorrect(); }, [state]); return ( <button ref={ref} className={`tap-button state-${state}`} onClick={onTap} disabled={state === "disabled"} > {label} </button> ); }
